ENGINEERING MINDSET

I'm a software engineer passionate about system design and the kind of problems that don't have obvious solutions. I gravitate toward architecture decisions, data modeling, and the mechanics of making things fast, reliable, and maintainable.

CDM CALCULATOR

Tool for finding the Minimum Dominating Set in large-scale graphs — an NP-Hard optimization problem. Implements both exact Backtracking with recursion-tree pruning and Greedy Heuristics for fast approximations. Includes an interactive graph editor built from scratch in Java AWT and a JUnit stress testing suite.

TECHNICAL CHALLENGE

Pruning the recursion tree aggressively enough to process dense graph instances that would otherwise exhaust memory. Balancing exact solutions (backtracking) vs. fast approximations (greedy) and exposing both through a visual, interactive interface.

◈

SPEC-FIRST DISCIPLINE

Every change starts with a proposal, then spec, then design. No code until the contract is clear — this prevents rework and scope creep.

⊞

ORCHESTRATION OVER EXECUTION

The AI plans and delegates; sub-agents execute in fresh contexts. Gates between phases catch drift before it compounds downstream.

⊿

ADVERSARIAL VERIFICATION

Judgment Day protocol: blind dual review, fix confirmed issues, re-judge. No rubber-stamp approvals.

◉

WORK-UNIT COMMITS

Each commit is a reviewable unit: code + tests + docs together. Chained PRs keep diffs focused and reviewers unburdened.

⬡

INCIDENT RECOVERY BUILT-IN

Wrong cwd, merge conflicts, test failures — mandatory fresh-context audit before continuing. No silent workarounds that hide root causes.
